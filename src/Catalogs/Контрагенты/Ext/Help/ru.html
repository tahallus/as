Справочник "Контрагенты"

Для хранения информации о покупателях и поставщиках используется справочник Контрагенты.

Редактирование информации о контрагентах производится в отдельном диалоговом окне, открываемом из формы списка справочника контрагентов.

При этом указывается информация, характеризующая контрагента:

  • Тип контрагента. Юридическое лицо, индивидуальный предприниматель, физическое лицо или государственный орган. Заполните доступные юр. реквизиты в зависимости от выбранного типа. Информация о контрагенте может быть заполнена автоматически по ИНН.
  • Вид. Вид взаимоотношений с контрагентом: покупатель, поставщик, прочие отношения. Установите нужный флажок "Покупатель" или "Поставщик" или оба флажка, если контрагент одновременно является покупателем и поставщиком. Для государственных организаций и банков установите флажок Прочие отношения. Покупатели будут доступны в разделе Продажи, Поставщики – в разделе Закупки, Прочие отношения – для перечисления налогов и взносов и взаимоотношений с банками.
  • Полное наименование контрагента. Используется для отражения в печатных формах документов.
  • Представление в программе. Название контрагента для отражения в информационной базе. Используется для выбора и быстрого поиска контрагента в списках.
  • Юридические реквизиты:
    • ИНН - идентификационный номер налогоплательщика представляет собой цифровой код, упорядочивающий сбор налогов с налогоплательщиков в Российской Федерации. ИНН присваивается как юридическим, так и физическим лицам.  Используется для отражения в печатных формах.
    • Головной контрагент (для юридических лиц) -Используется при формировании печатных форм, например, УПД или счет-фактура. Для отображения информации об иерархии по головному контрагенту на форме списка, включите поле "Головной контрагент" в подменю "Еще" - "Изменить форму"
    • КПП - код причины постановки на учет используется только для юридических лиц и является дополнением к ИНН в связи с постановкой на учет в разных налоговых органах. Используется для отражения в печатных формах документов.
    • ОКПО - код по общероссийскому классификатору предприятий и организаций присваивается в момент регистрации юридического лица и показывает вид деятельности. Используется для отражения в печатных формах документов.
    • Рег. номер - для иностранной организации можно указать регистрационный номер, присвоенный в стране регистрации
  • Адреса, телефоны. - контактная информация контрагента: E-mail, другое, почтовый адрес, телефон, факс, фактический адрес и юридический адрес.
  • Контакты. - информация о контактных лицах контрагента. Начните набирать фамилию или имя в строке и в выпадающем списке вам будут предложены для выбора подходящие варианты из справочника Контакты. Если нужного контакта в списке нет, после заполнения информации новый контакт будет добавлен в справочник Контакты и привязан к контрагенту. Один контакт может быть привязан к нескольким контрагентам. Если контактных лиц несколько, добавьте их по ссылке +контакт. По ссылке +телефон, адрес для каждого контакта можно указать дополнительную информацию – Skype, социальные сети, мессенджеры.
  • Группа. Все контрагенты могут быть распределены по группам, например Загрузка с сайта, Учредители, К обзвону. Кроме того, сами группы могут быть включены в другие группы, образуя вложенную иерархию.
  • #Теги. Отличительная метка для быстрого отбора контрагента в справочнике
  • Источник. Источник привлечения покупателя. Указывайте информацию об источнике и оценивайте, какой из каналов по привлечению клиентов работает эффективнее.
  • Заметки. Любая дополнительная информация о контрагенте.
  • Ответственный. Сотрудник ответственный за работу с контрагентом.
  • Статья. Статья движения денежных средств для автоматического заполнения в банковских и кассовых документах по этому контрагенту.
  • Группа доступа. Группа доступа контактов и контрагентов, определяющая ограничение доступа пользователей к элементам справочника и документам по контрагенту.
  • Учет взаиморасчетов. Группа реквизитов:
    • Аналитика взаиморасчетов с контрагентом. Выбирается в колонке "Долги": "по договорам", "по накладным" или "по заказам". Установка флага позволяет выводить в отчёте долги с аналитикой по указанному измерению. Также установка флага приводит к тому, что в документах требуется выбирать договор, документ или заказ.
    • Флажки в колонке "Оплата". Установка флажков позволяет видеть в списке заказов, счетов на оплату или накладных информацию об оплате в виде кружков.

При нажатии кнопки "Дополнительные реквизиты" открывается диалоговое окно редактирования дополнительных реквизитов, определяемых пользователем.

 

Заполнение реквизитов контрагента по ИНН

По кнопке "Заполнить по ИНН" происходит заполнение реквизитов контрагента. Заполнение производится на основе данных ЕГРЮЛ и ЕГРИП. При наличии соответствующей информации, будут заполнены следующие реквизиты:

для юридического лица:

- КПП;
- краткое и полное наименование;
- руководитель (будет создано контактное лицо);
- юридический адрес;
- фактический адрес (если не заполнен);
- телефон;

для физического лица (предпринимателя):
- краткое, полное наименование;
