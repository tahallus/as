Расчеты с поставщикам

Контроль расчетов с поставщиками выполняется следующим образом:
  • на дату окончания анализируемого периода формируется таблица остатков по регистру «Расчеты с поставщиками»;
  • полученная таблица анализируется и некорректной считается ситуация, когда у контрагента, по одному и тому же договору и заказу имеются остатки долгов и авансов.

Автоматическая корректировка выполняется по нажатию на гиперссылку «Выполнить зачет авансов». При этом происходит следующее:
  • из регистра взаиморасчетов выбираются документы, изменившие состояние расчетов, за текущий период;
  • в каждом из отобранных документов производится программное заполнение расшифровки платежа для документов, формирующих авансы и предоплат для документов, формирующих долги.

Для документов вида «Ввод начальных остатков» автоматическая корректировка не предусмотрена, т.е. если в рамках одного и того же договора будет внесен остаток авансов и долгов, то данная ситуация будет считаться некорректной. При нажатии на гиперссылку «Выполнить зачет авансов» данная ситуация автоматически обрабатываться не будет.

Курсовые разницы

Будет выполнена проверка движений по начислению курсовых разниц за учетный период.

В случае если будут обнаружены неадекватно большие курсовые разницы - пользователю будет выведено соответствующее сообщение.

 

Номенклатура без спецификаций

Контролируется, чтобы для каждой номенклатурной позиции, для которой создана одна или более спецификаций, была указана спецификация по умолчанию.

В случае обнаружения недочетов, пользователю предлагается:

  • в случае если для конкретной номенклатурной позиции создана одна спецификация –  установить её как спецификацию по умолчанию;
  • в случае если спецификаций больше чем одна – выбрать из списка необходимую.

Результаты исправлений могут влиять на результаты анализа подчиненных разделов. Влияние данного раздела на подчиненные будет описано отдельно для каждого из подчиненных разделов.

 

Отчеты переработчиков без спецификаций

Выполняется контроль того, чтобы во всех проведенных за анализируемый период документах вида «Отчет переработчика» была указана спецификация продукции.

В случае обнаружения незаполненных спецификаций выведены будут только те документы, для продукции которых указана спецификация по умолчанию. Поэтому важно выполнить все корректировки по разделу «Номенклатура без спецификаций». В выведенных документах пользователю предлагается установить спецификацию и перезаписать документы – выполняется по нажатию на гиперссылку «Установить спецификацию».

Результаты исправлений могут влиять на результаты анализа подчиненных разделов. Влияние данного раздела на подчиненные будет описано отдельно для каждого из подчиненных разделов.

 

Отчеты переработчиков - несоответствие списаний спецификациям

Выполняется контроль того, чтобы во всех проведенных за анализируемый период документах вида «Отчет переработчика» номенклатурный состав и количества табличной части «Запасы» соответствовали указанной спецификации. Контролироваться будут только документы, с указанной в них спецификацией продукции, поэтому выполнение корректировок по разделу «Отчеты переработчиков без спецификаций» важно для наиболее качественного анализа по данному разделу.

При нахождении несоответствий пользователю предлагается перезаполнить и перепровести каждый документ отдельно, по нажатию на гиперссылку «Перезаполнить и перепровести».

 

Документы «Производство» без спецификаций

Анализ закупочных цен

В ходе анализа будет выполнено сравнение закупочных цен в приходных накладных за анализируемый период с предыдущими поступлениями (приходными накладными за прошлый период). Проконтролированы будут только поступления номенклатуры с типом "Запас".

В случае, если будут обнаружены отклонения цен в 5 или более раз, пользователю будет выведен перечень «проблемных» номенклатурных позиций.

В данном разделе не предполагается каких-либо действий по автоматическому исправлению, но для каждой позиции доступен отчет, показывающий динамику изменения цен – по нажатию на гиперссылку «Результат анализа».

 

Контроли организаций и договоров в документах

В ходе анализа будет выполнена проверка совпадения договоров и организаций и документах проведенных за текущий период с договорами и организациями в подчиненных документах.

Подчиненными, в данном контексте, считаются те документы, ссылки на которые указаны в полях других документов.

Например:

  • если расходная накладная была введена на основании заказа покупателя, то заказ будет указан в поле «Основание» данной расходной, и при этом заказ будет считаться подчиненным расходной накладной;
  • при вводе приходной накладной в табличном поле «Запасы», содержащем 2 строки, были указаны 2 заказа поставщику – в данном случае оба заказа будут рассматриваться как подчиненные приходной накладной.

Кроме того, если настроен обмен с бухгалтерией, будет выполняться контроль совпадения организаций в документе и в договоре, выбранном в данном документе.

В случае обнаружения несоответствий пользователю будет выведен соответствующий отчет с перечнем документов и некорректных ситуаций.

 

Статьи движения денежных средств

В ходе анализа будет выполнена проверка соответствия движений денежных средств (платежей и поступлений ДС) предопределенным группам («Платежи», «Поступления») статей движения денежных средств за анализируемый период.

Алгоритм проверки следующий:

  • если некоторая статья ДДС входит в предопределенную группу «Платежи», то предполагается, что по данной статье могут быть только расходные движения ДС. Исключением считается ситуация возврата денег поставщиком (статья ДДС «Оплата поставщикам» и соответствующие виды операций в документах «Поступление в кассу» и/или «Поступление на счет») – в случае невыполнения описанных условий пользователю будет выведено сообщение о возможной ошибке учета;
  • если некоторая статья ДДС входит в предопределенную группу «Поступления», то предполагается, что по данной статье могут быть только приходные движения ДС. Исключением считается ситуация возврата денег покупателю (статья ДДС «Оплата от покупателей» и соответствующие виды операций в документах «Расход из кассы» и/или «Расход со счета») – в случае невыполнения описанных условий пользователю будет выведено сообщение о возможной ошибке учета.

Для каждой статьи ДДС, для которой были обнаружены неточности учета, можно будет просмотреть перечень документов движения за анализируемый период. Перечень документов будет отображен в виде отдельного отчета, вызвать который можно нажатием на конкретную статью.
