Документ "Платежное поручение"

Документ предназначен для формирования печатной формы банковских платежных поручений.

 

С помощью документа можно оформить следующие виды платежных поручений:

  • "Оплата",
  • "Перечисление в бюджет",
  • "На расходы",
  • "Зарплата по ведомости",
  • "Зарплата сотруднику"
  • "Подотчетнику"
  • "Прочему контрагенту (учредителю, таможне)",
  • "Покупателю (возврат)",
  • "Перевод на другой счет",
  • "Прочий расход",
  • "Личные средства".

Операция "Оплата" предназначена для формирования печатной формы платежного поручения на оплату контрагенту.

Если необходимо оформить операцию оплаты покупателю за возвращенные им ранее позиции номенклатуры, то устанавливается вид операции "Покупателю (возврат)". 

Операция "Перечисление в бюджет"  предназначена для формирования печатной формы платежного поручения на перечисление в бюджет.

Для оформления вывода личных средств со счета предпринимателя используется вид операции "Личные средства".

Для оформления расходования денежных средств на выплату заработной платы по ведомости устанавливается вид операции "Зарплата по ведомости".

Для оформления расходования денежных средств на выплату заработной платы сотруднику без ведомости устанавливается вид операции "Зарплата сотруднику".

В случае оформления операции выдачи денежных средств подотчетнику устанавливается вид операции "Подотчетнику".

Для формирования печатной формы нужно для любого вида операции заполнить следующие реквизиты:

  • Организация, от которой выписывается платежное поручение. Выбирается из справочника "Организации". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по нескольким организациям.
  • Вид операции. Указывается вид операции документа. Предлагается выбор из списка: оплата или перечисление в бюджет.
  • Банковский счет организации, для которого оформляется платежное поручение.
  • Получатель платежа. Выбирается из справочника "Контрагенты".
  • Счет получателя платежа. Выбирается из справочника "Банковские счета".
  • Очередность платежа.
  • Оплачено - признак оплаты платежного поручения.
  • Вид платежа. Предлагается выбор из списка: электронно, почтой, телеграфом, срочно.
  • ИНН/КПП плательщик. Группа реквизитов:
    • ИНН. Идентификационный номер налогоплательщика плательщика.
    • КПП. Код причины постановки на учет плательщика.
    • Плательщик. Отображается в соответствующем поле печатной формы документа.
  • ИНН/КПП получатель. Группа реквизитов:
    • ИНН. Идентификационный номер налогоплательщика получателя платежа.
    • КПП. Код причины постановки на учет получателя платежа.
    • Получатель. Отображается в соответствующем поле печатной формы документа.
  • Сумма. Указывается сумма платежа.
  • Ставка НДС. Выбирается из справочника "Ставки НДС".
  • Сумма НДС. Указывается сумма НДС.
  • Назначение платежа. Текст назначения платежа. Отображается в соответствующем поле печатной формы документа.
  • Комментарий. Указывается комментарий к документу.
 
Для вида операции "Перечисление в бюджет" становится доступной для заполнения закладка "Перечисление в бюджет", на которой нужно заполнить следующие реквизиты:
  • Вид перечисления в бюджет. Указывается вид перечисления в бюджет: "Налоговый платеж", "Таможенный платеж", "Иной платеж".
  • Статус составителя. Указывается статус составителя платежного документа на перечисление налогов, сборов и иных обязательных платежей.
  • КБК. Указывается код бюджетной классификации доходов РФ.
  • Код ОКТМО - указывается код территории (населенного пункта), на которой мобилизуются средства.
  • УИН - уникальный идентификатор начисления. Если не предоставлен получателем денег, то не указывается.
  • Основание платежа. Указывается условное обозначение основания налогового платежа.
  • Налоговый период. Указывается условное обозначение периода, за который перечисляется налог.
  • Номер документа. Указывается номер документа - основания перечисления  налога, сбора или иного платежа.
  • Дата документа. Указывается дата документа - основания перечисления  налога, сбора или иного платежа.
  • Тип платежа. Указывается условное обозначение типа налогового платежа.

 

Для вида операции "Оплата" или "Покупателю (возврат)", становятся видимыми следующие реквизиты:

  • Расшифровка платежа. Табличная часть:
    • Статья движения денежных средств. Выбирается из справочника "Статьи движения денежных средств".
    • Аванс. Признак аванса платежа.
    • Документ отгрузки (Документ расчетов), по которому осуществляется оплата. Для вида операции "Поставщику" указывается только для платежей не относящихся к авансам. Для вида операции "Покупателю" указывается всегда.
    • Сумма расчетов. Сумма оплаты. Указывается в валюте расчетов по договору с контрагентом.
    • Курс валюты расчетов по договору с контрагентом. Реквизит отображается на форме, если в настройках параметров учета включено ведение учета валютных операций.
    • Кратность валюты расчетов по договору с контрагентом. Реквизит отображается на форме, если в настройках параметров учета включено ведение учета валютных операций.
    • Сумма платежа. Сумма оплаты. Указывается в валюте расчетного счета. Реквизит отображается на форме, если в настройках параметров учета включено ведение учета валютных операций.
    • Ставка НДС. Выбирается из справочника "Ставки НДС".
    • Сумма НДС. Указывается сумма НДС.
    • Оплачиваемый счет. Указывается в случае оплаты по определенному счету. Выбирается из списка документов "Счет на оплату поставщика".
    • Оплачиваемый заказ. Указывается в случае осуществления платежа по определенному заказу.
    • Документ планирования платежа. Выбирается из списка документов "Перемещение денег (план)" или Поступление денег (план)".

 

Для вида операции "Зарплата по ведомости" становятся видимыми следующие реквизиты:

  • Направление деятельности, отнесения расходов по кассовому методу учета доходов и расходов. Выбирается из справочника "Направления деятельности". Реквизит отображается на форме, если в настройках параметров учета включено использование кассового метода учета доходов и расходов.
  • Выплата заработной платы. Табличная часть:
    • Ведомость. Платежная ведомость, по которой выплачиваются денежные средства. Выбирается из списка документов "Платежные ведомости".
    • Сумма платежа. Сумма выплаты заработной платы по ведомости. Указывается в валюте кассы.
    • Документ планирования платежа. Выбирается из списка документов "Перемещение денег (план)" или "Заявка на расход денег (план)".

В соответствии с Указанием Банка России от 14.10.2019 №5286-У для операций «Зарплата сотруднику» и «Прочему контрагенту» становится видим реквизит "Удержание по исполнительному листу". Реквизит выводится, если дата документа больше или равна дате вступления закона в силу (01.06.2020). Для операции "Прочему контрагенту" реквизит видим, если контрагент является физическим лицом.


Для операций «Зарплата по ведомости», «Зарплата сотруднику» и «Прочий расход» становится видим реквизит "Код выплат". Для операции "Прочему контрагенту" реквизит видим, если контрагент является физическим лицом.

Код выплат выводится в поле 110 печатной формы платежного поручения.
Указывает на то, что выплата является государственным пособием.
Государственное пособие за счет средств бюджета, предусмотренное частями 5.5 и 5.6 статьи 30.5 Федерального закона № 161-ФЗ, переводится на карту "Мир":

  - пособие по беременности и родам

  - единовременное пособие женщинам, вставшим на учет в медицинских организациях в ранние сроки беременности

  - единовременное пособие при рождении ребенка

  - ежемесячное пособие по уходу за ребенком


Документ "Платежное поручение" можно подписать и отправить в банк прямо из "1С:Управление нашей фирмой".

Такая возможность доступна при включении прямого обмена электронными документами с банками в настройках обмена электронными документами.

Состояние платежного документа может принимать следующие значения:

  • Не сформирован - для платежного поручения обмен электронными документами не проводился.
  • На подписи - электронный документ сформирован и готов к подписи.
  • Ожидается отправка - электронный документ имеет все необходимые подписи и готов к отправке в банк.
  • Ожидается подтверждение - платежное поручение отправлено в банк, но подтверждение о его исполнении еще не поступило.
  • Платеж исполнен - поступило подтверждение об исполнении платежного поручения.
  • Требуется исправление - электронный документ содержит ошибки, либо при его отправке возникли ошибки.
  • Отклонен пользователем - электронный документ отклонен пользователем до отправки в банк.
 
Документ "Платежное поручение" может быть введен на основании документов:

  • "Заказ поставщику";
  • "Счет на оплату поставщика";
  • "Приходная накладная";
  • "Заявка на расход денег";
  • "Авансовый отчет".
