Документ "Отчет комиссионера"

Документ предназначен для:

  • Хранения информации о регистрации факта продажи комиссионером запасов, которые были переданы нашей организацией ему на комиссию (на реализацию).

  • Хранения информации о регистрации факта возврата товаров покупателем комиссионеру, по которым комиссионером уже был создан документ "Отчет комиссионера" с операцией "Продажа".

 

В документе реализовано две операции:


  • Продажа. Соответствует хозяйственной операции "Отчет комиссионера о продажах". Фиксирует  факт продажи комиссионером запасов. 

  • Возврат. Соответствует хозяйственной операции "Отчет комиссионера о возвратах". Фиксирует  факт возврата товаров покупателем комиссионеру .


Необходимо заполнить следующие реквизиты:

  • Организация. В данном реквизите указывается собственная организация, от имени которой оформляется документ. Выбирается из справочника "Организации". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по нескольким организациям.

  • Контрагент. В данном реквизите указывается комиссионер. Выбирается из справочника "Контрагенты".

  • Договор. В данном реквизите указывается договор контрагента, в рамках которого оформляется документ. Выбирается из справочника "Договоры контрагентов".

 

На форме документа расположена гиперссылка "Редактировать цены и валюту", по которой вызывается диалог редактирования видов цен и валюты данного документа.

  • Дополнительно. Группа реквизитов:

    • Способ расчета комиссионного вознаграждения. Указывается способ расчета комиссионного вознаграждения. Выбирается из следующих вариантов:

      • Не рассчитывается (такой способ расчета означает, что сумма комиссионного вознаграждения указывается индивидуально).

      • Процент от суммы продажи (под суммой продаж понимается та сумма, по которой комиссионер продал запасы).

      • Процент от разности сумм продажи и поступления (под суммой продаж понимается та сумма, по которой комиссионер продал запасы, а под суммой поступления понимается та сумма, по которой запасы были переданы ему на реализацию).

    • Процент комиссионного вознаграждения. Указывается процент для расчета комиссионного вознаграждения.

    • Ставка НДС с комиссионного вознаграждения. Указывается ставка налога на добавленную стоимость с комиссионного вознаграждения.

    • Удержать комиссионное вознаграждение. Указывается признак удержания комиссионного вознаграждения.

  • Покупатели. В данной табличной части комитент регистрирует информацию о покупателях и датах реализации. При записи документа по строкам, в которых установлен флаг "Выписан счет-фактура" автоматически создаются счета-фактуры.

    • Покупатель - покупатель запасов комитента. Выбирается из справочника "Контрагенты".

    • Всего - стоимость запасов, проданных комитентом покупателю. Реквизит рассчитывается автоматически на основании данных табличной части "Запасы".

    • Выставлен СФ - признак необходимости автоматической регистрации счет-фактуры. (для операции "Продажа")

    • Дата СФ - дата реализации покупателю запасов комитента. (для операции "Продажа")

    • Счет-фактура - ссылка на выставленный счет-фактуру. (для операции "Продажа")

  • Запасы. Данная табличная часть подчиняется табличной части "Покупатели". Одной строке табличной части "Покупатели" может соответствовать несколько строк табличной части "Запасы". Табличная часть "Запасы" может быть заполнена с помощью подбора (кнопка "Подбор" командной панели табличной части) либо методом ручного ввода. Реквизиты табличной части:

    • Номенклатура - позиция номенклатуры, за реализацию которой комиссионер отчитывается перед нашей организацией. Выбирается из справочника "Номенклатура" с типом "Запас".

    • Характеристика - при необходимости в данном поле указывается характеристика номенклатуры. Выбирается из справочника "Характеристики номенклатуры". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по характеристикам номенклатуры.

    • Партия - при необходимости в данном поле указывается партия номенклатуры. Выбирается из справочника "Партии номенклатуры". Реквизит отображается на форме, если в настройках параметров учета включено ведение партионного учета.

    • Количество - количество реализованной номенклатуры, за которую отчитывается комиссионер.

    • Ед. - единица измерения номенклатурной позиции. Выбирается из справочника "Классификатор единиц измерения" или "Единицы измерения".

    • Цена - цена, по которой номенклатурная позиция была реализована комиссионером.

    • Сумма - стоимость номенклатуры по данной строке.

    • Ставка НДС - ставка налога на добавленную стоимость. Выбирается из справочника "Ставки НДС".

    • Сумма НДС - сумма налога на добавленную стоимость по данной строке.

    • Всего - поле, автоматически заполняемое суммарной стоимостью запасов, которые были реализованы комиссионером, согласно настройкам цен и валют. Настройки устанавливаются в диалоге, вызываемом по гиперссылке "Редактировать цены и валюту".

    • Страна происхождения - страна происхождения запасов (см. опцию "Прямое списание ГТД").

    • Номер ГТД - грузовая таможенная декларация запаса (см. опцию "Прямое списание ГТД").

    • Прямое списание ГТД - опция указывает, как списывать запасы по номерам ГТД. Если опция включена, остатки по номерам ГТД будут списаны с регистра "Остатки в разрезе ГТД". Если опция выключена, то предполагается, что данные по импорту были указаны при передачи на комиссию и, соответственно, программа будет закрывать регистр "Остатки в разрезе ГТД (переданные)".

    • Цена передачи - цена, по которой номенклатурная позиция была передана комиссионеру.

    • Сумма передачи - стоимость, по которой номенклатурная позиция была передана комиссионеру. 

    • Сумма НДС передачи - сумма налога на добавленную стоимость с суммы передачи по данной строке.

    • Сумма вознаграждения - сумма комиссионного вознаграждения, рассчитанная исходя из условий договора с комиссионером и параметров закладки "Дополнительно".

    • Сумма НДС вознаграждения - сумма налога на добавленную стоимость с суммы вознаграждения по данной строке.

    • Заказ - заказ покупателя, в рамках которого передавались запасы комиссионеру. Выбирается документ "Заказ покупателя". Реквизит необязателен для заполнения.

  • Предоплата. Данная табличная часть заполняется сведениями по зачету авансов, полученных ранее от комиссионера. На форме документа расположена гиперссылка "Редактировать зачет предоплаты", по которой вызывается диалог редактирования табличной части "Зачет предоплаты". Реквизиты табличной части:

    • Документ - документ поступления аванса от покупателя: "Поступление в кассу", "Поступление на счет".

    • Заказ - здесь можно сослаться на документ "Заказ покупателя".

    • Сумма расчетов - сумма в валюте договора.

    • Курс - курс пересчета в валюту документа.

    • Кратность - кратность пересчета в валюту документа.

    • Сумма платежа - указывается в валюте документа.

  • Дополнительная информация. Группа реквизитов:

    • Подразделение - информация о подразделение, ответственном за операцию. Выбирается из справочника "Структурные единицы" с типом "Административно-управленческое подразделение". Реквизит необязателен для заполнения.

    • Ответственный - информация о сотруднике, ответственном за ввод текущего документа. Выбирается из справочника "Сотрудники". Реквизит необязателен для заполнения.

    • Комментарий - поле ввода, предназначенное для указания любой дополнительной информации к документу.
