Документ "Акт выполненных работ"

Документ предназначен для хранения информации по отражению в учете реализации работ и услуг сторонним контрагентам.

 

Необходимо заполнить следующие реквизиты:

  • Организация. В данном реквизите указывается собственная организация, в которой выполняется операция. Выбирается из справочника "Организации". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по нескольким организациям.

  • Основание. В данном реквизите указывается "Счет на оплату", на основании которого сформирован данный документ. Реквизит не обязателен для заполнения.

  • Заказ покупателя. Выбирается документ "Заказ покупателя". Есть возможность сменить расположение реквизита с помощью команды "Шапка / Табличная часть". Реквизит не обязателен для заполнения.

  • Заказ на производство. Выбирается документ "Заказ на производство". Есть возможность сменить расположение реквизита с помощью команды "Шапка / Табличная часть". Реквизит не обязателен для заполнения.

  • Контрагент. В данном реквизите указывается покупатель. Выбирается из справочника "Контрагенты".

  • Договор. В данном реквизите указывается договор контрагента, в рамках которого оформляется документ. Выбирается из справочника "Договоры контрагентов".

  • Подразделение. Выбирается из справочника "Структурные единицы" с типом "Подразделение".

 

На форме документа расположена гиперссылка "Редактировать цены и валюту", по которой вызывается диалог редактирования видов цен, скидок и валюты данного документа.

В подвале формы можно указать ручную скидку на документ, процентом или суммой.

  • Работы и услуги. Данная табличная часть может быть заполнена с помощью подбора (кнопка "Подбор" командной панели табличной части), сканированием штрихкода либо методом ручного ввода. Реквизиты табличной части:

    • Номенклатура - позиция номенклатуры, которая реализуется. Выбирается из справочника "Номенклатура" с типом "Услуга" или "Работа".

    • Характеристика - при необходимости в данном поле указывается характеристика реализуемой номенклатуры. Выбирается из справочника "Характеристики номенклатуры". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по характеристикам номенклатуры.

    • Количество - количество реализуемой номенклатурной позиции.

    • Ед. - единица измерения номенклатурной позиции. Выбирается из справочника "Классификатор единиц измерения" или "Единицы измерения".

    • Цена - цена реализуемой номенклатурной позиции.

    • % Скидки - процент ручной скидки (наценки).

    • % Авт. - процент автоматической скидки (наценки). Процент автоматической скидки редактируется в справочнике "Виды автоматических скидок и наценок". Расчёт автоматических скидок в документе осуществляется при записи документа и при нажатии кнопки "% Авт." над табличной частью.

      Сумма - стоимость реализуемой номенклатуры по данной строке.

    • Ставка НДС - ставка налога на добавленную стоимость. Выбирается из справочника "Ставки НДС".

    • Сумма НДС - сумма налога на добавленную стоимость по данной строке.

    • Всего - поле, автоматически заполняемое суммарной стоимостью услуг или работ согласно настройкам цен и валют. Настройки устанавливаются в диалоге, вызываемом по гиперссылке "Редактировать цены и валюту".

    • Заказ покупателя. Выбирается документ "Заказ покупателя". Есть возможность сменить расположение реквизита с помощью команды "Шапка / Табличная часть". Реквизит не обязателен для заполнения.

    • Заказ на производство. Выбирается документ "Заказ на производство". Есть возможность сменить расположение реквизита с помощью команды "Шапка / Табличная часть". Реквизит не обязателен для заполнения.

    • Содержание - представление работы (услуги) в печатной форме. Чтобы реквизит стал видимым, необходимо выполнить "Все действия" - "Изменить форму" и установить флаг напротив реквизита.

  • Предоплата. Данная табличная часть заполняется сведениями по зачету авансов, полученных ранее от покупателя. На форме документа расположена гиперссылка "Редактировать зачет предоплаты", по которой вызывается диалог редактирования табличной части "Зачет предоплаты". Реквизиты табличной части:

    • Документ - документ поступления аванса от покупателя: "Поступление в кассу", "Поступление на счет".

    • Заказ - здесь можно сослаться на документ "Заказ покупателя".

    • Сумма расчетов - сумма в валюте договора.

    • Курс - курс пересчета в валюту документа.

    • Кратность - кратность пересчета в валюту документа.

    • Сумма платежа - указывается в валюте документа.

  • Дополнительная информация. Группа реквизитов:

    • Ответственный - информация о сотруднике, ответственном за ввод текущего документа. Выбирается из справочника "Сотрудники". Реквизит не обязателен для заполнения.

    • Комментарий - поле ввода, предназначенное для указания любой дополнительной информации к документу.

    • Подпись кассира - поле доступно для вида операции "Заказ-наряд". Если поле пустое, то при печати чека ККМ из формы документа, выводится ФИО по данным автора документа. Если поле заполнено, то при печати чека ККМ из формы документа, выводится ФИО по данным подписи кассира, которое выбрано в поле.

 Минимальные цены


При включении опции "Минимальные цены продажи" и настройки минимальных цен (Раздел "Продажи" -> "Еще больше возможностей") в табличной части цвет текста поля цена будет изменен на красный, если значение цены станет меньше чем минимальная.
Минимальные цены будут подбираться из общего вида минимальных цен.

Акт выполненных работ может быть введен на основании документов:

  • "Заказ покупателя",

  • "Счет на оплату".
